Phase 1

Introduction du Projet

Ce projet vise à prédire le taux de désabonnement (Churn) des clients de Horizon Telecom en utilisant des techniques avancées d'apprentissage automatique. L'objectif est d'identifier les profils à risque pour permettre des actions de fidélisation proactives.

Phase 2

Exploration & Nettoyage

Analyse approfondie de la structure des données et traitement des valeurs manquantes. La colonne TotalCharges a été convertie en format numérique et les espaces vides ont été gérés.

# Conversion de TotalCharges et suppression des IDs df['TotalCharges'] = pd.to_numeric(df['TotalCharges'], errors='coerce') df.dropna(inplace=True) df.drop('customerID', axis=1, inplace=True)

Nous avons observé un déséquilibre des classes : environ 26% de Churn contre 74% de clients fidèles.

Phase 3

Prétraitement & Pipeline

Utilisation de LabelEncoder pour les variables catégorielles et StandardScaler pour la normalisation des données numériques (Tenure, Charges).

scaler = StandardScaler() numeric_cols = ['tenure', 'MonthlyCharges', 'TotalCharges'] df[numeric_cols] = scaler.fit_transform(df[numeric_cols])

Les 10 caractéristiques les plus pertinentes ont été sélectionnées via SelectKBest.

Phase 4

Équilibrage avec SMOTEENN

Pour corriger le déséquilibre des classes, nous avons appliqué la technique SMOTEENN (Combining SMOTE and Edited Nearest Neighbors), ce qui a considérablement amélioré le rappel (Recall) pour la classe minoritaire.

from imblearn.combine import SMOTEENN smote_enn = SMOTEENN() X_resampled, y_resampled = smote_enn.fit_resample(X, y)
Phase 5

Comparaison des Modèles

Plusieurs algorithmes ont été testés et optimisés via RandomizedSearchCV :

  • Logistic Regression : Modèle de base (Baseline).
  • Decision Tree : Bonne interprétabilité mais tendance au surapprentissage.
  • Random Forest : Meilleur compromis entre précision et généralisation.
  • Gradient Boosting : Hautes performances mais plus lent à entraîner.
